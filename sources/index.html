Public reference · agent gating logic

Source contracts.

28 datasets, each with a published freshness SLA. Fresh = age < SLA. Degraded = SLA ≤ age < 2×SLA. Stale = age ≥ 2×SLA.

Every /api/data/:name response includes a provenance field with observed_at, age_seconds, and freshness_status. Agents should gate on status before consuming.

Reading provenance from /api/data

Each dataset response includes:

{
  "dataset": "yahoo-quotes",
  "date": "2026-05-19",
  "data": { ... },
  "provenance": {
    "source": "yahoo-quotes",
    "observed_at": "2026-05-19T06:00:00.000Z",
    "age_seconds": 420,
    "sla_seconds": 900,
    "freshness_status": "fresh"
  }
}

Defensive agent pattern: skip the dataset if freshness_status === "stale"; degrade gracefully (warn, use fallback, or proceed with annotation) if "degraded"; consume freely if "fresh".

If you spot drift between published SLA and observed behavior, that is a bug — file at github.com/MrCerq/agentcanary-backend/issues.
